01

Light application surfaces

Implemented + refined
Brandon's words“Use #D1D5DB for the site-wide page background and keep working cards/surfaces white.”

What was implementedThe requested light canvas and white working surfaces were implemented. After the first pass was reviewed, the flat gray was refined to #EEF3F8, with white primary surfaces and a restored dark navy header/Admin rail for stronger visual hierarchy.

Final-state note: the exact initial gray was superseded by Andrew's direction to produce a better light mode; the underlying requirement for a light canvas and white work surfaces remains intact.
02

Button hover contrast

Implemented
Brandon's words“Increase button hover contrast site-wide.”

What was implementedA global hover treatment now changes brightness and saturation on every enabled button. Buttons inside the dark header and Admin rail receive a brighter hover response calibrated for those darker surfaces.

03

Simplified primary navigation

Implemented
Brandon's words“Simplify the header to separate Dashboard, Customers, Deals, Inventory, Schedule, Inbox, Citadel, and Reports tabs; remove CRM, Sales, and Service group labels; rename Comms to Inbox.”

What was implementedThe grouped CRM, Sales, and Service labels were removed. All eight destinations are now independent, plainly named tabs, including Inbox in place of Comms. The active tab is emphasized while inactive tabs remain quiet.

04

Dashboard parts label

Implemented
Brandon's words“Rename Dashboard ‘Parts Overdue’ to ‘Parts On Order’ without changing its underlying count.”

What was implementedThe dashboard tile now reads Parts On Order. It still uses the existing overduePartsCount value, so only the customer-facing label changed.

05

One customer list

Implemented
Brandon's words“Customers must be one list with search and sort controls; remove Prospect, Lead, and Past Customer filter tabs; default sort is Recent activity.”

What was implementedThe lifecycle filter tabs were removed and List view is the default. Search covers name, phone, and email; Recent activity remains the default sort. The existing customer-card view and Cards/List selector were preserved as an optional user choice because they do not conflict with the requested default.

06

Active Deals terminology

Implemented
Brandon's words“Rename Deals ‘Open Pipeline’ to ‘Active Deals.’”

What was implementedThe Deals summary tile now reads Active Deals. The existing pipeline value and active-deal count remain unchanged.

07

Overdue Sales Tasks terminology

Implemented
Brandon's words“Rename Deals ‘Closing This Week’ to ‘Overdue Sales Tasks.’”

What was implementedThe Deals tile now reads Overdue Sales Tasks. The existing count and value source were preserved.

08

Inventory brand filtering

Implemented
Brandon's words“Add an Inventory brand dropdown for All Brands, Sundance Spas, Master Spas, Platinum Spas, and Eco Spas.”

What was implementedA brand dropdown was added with the five requested choices. All Brands shows the full inventory; each named brand filters the list against the inventory item's brand value.

Service status headers

What was implementedThe Schedule header now exposes only Delivery, Warranty, and Parts On Order. The change is limited to the visible legend/filter headers; underlying service jobs and statuses were not deleted or rewritten.
